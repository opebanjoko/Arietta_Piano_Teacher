# Gate App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A throwaway guided browser harness at `spike/gate/`, deployed as Railway service `gate`, that walks the operator through both gate runbooks: prompt → record → detector verdict → attest → WAV auto-download → tallies → exportable gate record.

**Architecture:** Plain ES modules, no build step. Recorded takes are analyzed offline with the exact CI scoring functions (`processBuffer`, `processBufferPoly` from `spike/lib/pipeline.js`). Pure modules (tasks, analyze, wav, zip) are node-tested; capture and UI are browser-only and verified with Playwright through a `window.__gateStream` seam.

**Tech Stack:** Existing `spike/lib/*` + `spike/worklet.js`. Zero dependencies.

**Spec:** `docs/superpowers/specs/2026-07-11-gate-app-design.md` — read it first.

## Global Constraints

- Throwaway code, but honest math: verdicts must call the same pipeline functions CI uses — never a reimplementation.
- Filenames follow the runbooks verbatim: mono `<instrument>-<note>-<distance>-<take>.wav`, poly `poly-<piano>-<chord>-<distance>-<take>.wav`.
- Mic constraints match the product: `channelCount:1, echoCancellation:false, autoGainControl:false`.
- No emojis in code/logs. Spike style: semicolons. Downloads only ever fire from a click handler (Safari).
- Poly chord tasks stay within detector range (fundamentals MIDI 48–72); G4 major triad is excluded by design.
- Noise soak keeps no audio. Strike-to-pixel latency (G3) is out of scope and said so on the results screen.

## File Structure

```

spike/gate/gate.js      wizard state machine, screens, downloads, localStorage
```

```js
// spike/gate/wav.js
/** Float32 mono samples -> 16-bit PCM WAV bytes. */
export function encodeWav(samples, sampleRate) {
  const out = new Uint8Array(44 + samples.length * 2);
  const dv = new DataView(out.buffer);
  const str = (o, s) => { for (let i = 0; i < s.length; i++) out[o + i] = s.charCodeAt(i); };
  str(0, 'RIFF'); dv.setUint32(4, 36 + samples.length * 2, true); str(8, 'WAVE');
  str(12, 'fmt '); dv.setUint32(16, 16, true); dv.setUint16(20, 1, true); dv.setUint16(22, 1, true);
  dv.setUint32(24, sampleRate, true); dv.setUint32(28, sampleRate * 2, true);
  dv.setUint16(32, 2, true); dv.setUint16(34, 16, true);
  str(36, 'data'); dv.setUint32(40, samples.length * 2, true);
  for (let i = 0; i < samples.length; i++) {
    const s = Math.max(-1, Math.min(1, samples[i]));
    dv.setInt16(44 + i * 2, s < 0 ? s * 32768 : s * 32767, true);
  }
  return out;
}
```

```js
// spike/gate/zip.js
/** Minimal stored-entry ZIP (WAVs do not compress). */
const TABLE = (() => {
  const t = new Uint32Array(256);
  for (let n = 0; n < 256; n++) {
    let c = n;
    for (let k = 0; k < 8; k++) c = c & 1 ? 0xedb88320 ^ (c >>> 1) : c >>> 1;
    t[n] = c >>> 0;
  }
  return t;
})();

export function crc32(bytes) {
  let c = 0xffffffff;
  for (let i = 0; i < bytes.length; i++) c = TABLE[(c ^ bytes[i]) & 0xff] ^ (c >>> 8);
  return (c ^ 0xffffffff) >>> 0;
}

export function encodeZip(entries) {
  const enc = new TextEncoder();
  const locals = []; const centrals = [];
  let offset = 0;
  for (const { name, data } of entries) {
    const n = enc.encode(name); const crc = crc32(data);
    const local = new Uint8Array(30 + n.length + data.length);
    const dv = new DataView(local.buffer);
    dv.setUint32(0, 0x04034b50, true); dv.setUint16(4, 20, true);
    dv.setUint32(14, crc, true); dv.setUint32(18, data.length, true); dv.setUint32(22, data.length, true);
    dv.setUint16(26, n.length, true);
    local.set(n, 30); local.set(data, 30 + n.length);
    const central = new Uint8Array(46 + n.length);
    const cd = new DataView(central.buffer);
    cd.setUint32(0, 0x02014b50, true); cd.setUint16(4, 20, true); cd.setUint16(6, 20, true);
    cd.setUint32(16, crc, true); cd.setUint32(20, data.length, true); cd.setUint32(24, data.length, true);
    cd.setUint16(28, n.length, true); cd.setUint32(42, offset, true);
    central.set(n, 46);
    locals.push(local); centrals.push(central);
    offset += local.length;
  }
  const cdStart = offset; let cdLen = 0;
  for (const c of centrals) cdLen += c.length;
  const eocd = new Uint8Array(22);
  const ev = new DataView(eocd.buffer);
  ev.setUint32(0, 0x06054b50, true); ev.setUint16(8, entries.length, true); ev.setUint16(10, entries.length, true);
  ev.setUint32(12, cdLen, true); ev.setUint32(16, cdStart, true);
  const out = new Uint8Array(offset + cdLen + 22);
  let p = 0;
  for (const b of [...locals, ...centrals, eocd]) { out.set(b, p); p += b.length; }
  return out;
}
```

### Task 3: Analysis verdicts

**Files:** Create `spike/gate/analyze.js`; extend `spike/test/gate.test.js`.

**Interfaces:**
- Consumes: `processBuffer`, `processBufferPoly` from `../lib/pipeline.js`; `midiToName` from `../lib/pitch.js`.
- Produces: `analyzeTake(samples, sampleRate, task) -> { heard: string, match: boolean, events }`:
  - mono task (`expect.midi`): run `processBuffer` (mpm); heard = unique pitches in order; match iff unique pitch list === `[expect.midi]`.
  - chord task (`expect.midis`): run `processBufferPoly`; match iff exactly one event whose `midis` equal the expected set (extra events allowed only if subsets of it — ring-out re-strikes are operator retakes anyway; keep strict: exactly one event, equal set).
  - sequence task (`expectSeq`): run `processBufferPoly`; match iff the emitted `midis` arrays equal `expectSeq` in order.
  - `heard` is a human string like `"C4"`, `"C4 + E4 + G4"`, or `"C3 | C4 | D4 | ..."`.

```js
test('analyzeTake verdicts: mono right, mono wrong, chord, sequence', () => {

    { midis: [48], atMs: 200, durMs: 3000 },
    { midis: [60], atMs: 900, durMs: 600 },
    { midis: [62], atMs: 1600, durMs: 600 }
```

### Task 4: Capture, wizard UI, downloads

**Files:** Create `spike/gate/capture.js`, `spike/gate/gate.js`, `spike/gate/index.html`.

**Interfaces:**
- `capture.js`: `createCapture() -> { start(): Promise<{sampleRate}>, record(sec): Promise<Float32Array>, live(onChunk, chunkSec): () => void, stop() }`.
  - `start()` uses `window.__gateStream ?? getUserMedia({ audio: { channelCount: 1, echoCancellation: false, autoGainControl: false, noiseSuppression: false } })`, `AudioContext` + `audioWorklet.addModule('../worklet.js')`, worklet name `'capture'`, message shape `{ frame: Float32Array(2048), tMs }` with hop 1024 — reconstruct contiguous audio by keeping the first frame whole, then `frame.subarray(1024)` per subsequent message.
  - `record(sec)` resolves once `sec * sampleRate` samples accumulate. `live(onChunk, chunkSec)` calls back with each contiguous chunk (for live check / soak) and returns an unsubscribe.
- `gate.js` screens: `setup` (piano text/select, distance select, session buttons) → `corpus` (prompt card, Record button with countdown, verdict card with Yes / Retake / Played-something-else, progress `k of n`) → `live` (Start/Stop, rolling detections list each with Right/Wrong buttons, tally) → `soak` (Start, mm:ss countdown, event counter, pass/fail line) → `results` (tallies per session, Download all WAVs (zip), Download gate record (markdown), Download raw JSON, note that strike-to-pixel latency stays on the camera runbook).
  - State: `{ config, sessions: { mono: { attested: [{id, filename, match, human}] }, poly: {...}, live: { right, wrong }, soak: { events, minutes } } }` in `localStorage['arietta-gate']`. WAV bytes kept in-memory (`Map<filename, Uint8Array>`) for the zip; every attested take also downloads immediately.
  - Downloads: `Blob` + `URL.createObjectURL` + programmatic `<a download>` click, always inside the attest/export click handler.
  - Live check chunking: analyze 1.5 s contiguous chunks with `analyzeTake`-style pipeline calls (`processBufferPoly` when a "chords" toggle is on, else `processBuffer`), list any events for attestation.
  - Soak: same chunked analysis (both detectors' note events count), increment counter per event; no audio retained.
  - Gate-record markdown: rows for POLY_GATE_RUNBOOK's table (P2 corpus detection, P3 live, P3 soak) and GATE_RUNBOOK equivalents, computed from tallies, plus date and config.
- `index.html`: single page, warm minimal CSS (Arietta-ish cream/ink palette), `<main id="app">`, `<script type="module" src="./gate.js">`.
